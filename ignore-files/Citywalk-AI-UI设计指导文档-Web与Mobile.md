# Cultural Citywalk UI 设计指导文档

> 适用范围：Web 桌面端 + 手机端  
> 视觉方向：文化杂志 × 文学地图 × 城市档案  
> 核心气质：克制、安静、理性、有文化感  
> 产品关键词：城市阅读、主题漫游、路线编辑、时间切片、地图、档案、书页

---

# 1. 设计定位

Cultural Citywalk 不应被设计成：

- 传统旅游攻略网站
- 地图导航工具
- 小红书式卡片流
- ChatGPT 式纯聊天页面
- 手机 App 界面直接放大到网页

它应该更像：

> 一本可以边走边读的城市杂志。

或者：

> 一个帮助用户规划、阅读和保存城市漫游路线的数字出版物。

整体参考方向：

- Monocle
- Kinfolk
- Cereal
- Financial Times Weekend
- The New Yorker
- 文学地图集
- 图书馆目录卡
- 档案馆索引系统
- Medium / Apple Books 的阅读体验

---

# 2. 核心视觉原则

## 2.1 地图是工具，内容是主角

地图不应长期占据页面主体。

默认状态：

- 路线内容优先
- 地图折叠或窄栏显示
- 用户需要导航时再展开地图

桌面端：

- 阅读模式：地图约 20%～30%，内容约 70%～80%
- 导航模式：地图约 60%～70%，内容约 30%～40%

手机端：

- 阅读模式：地图默认折叠
- 导航模式：地图全屏或占据页面上半部

## 2.2 主题是“滤镜”，不是孤立频道

历史、文学、建筑、音乐、书店等主题应作为路线的增强层。

UI 上可使用主题标签：

```text
历史   文学   建筑   音乐   书店
```

需要区分两类操作：

- 仅切换讲解角度
- 同时优化路线站点

不应让用户误以为切换主题会完全重做路线。

## 2.3 像编辑一本杂志，而不是操作后台系统

界面应有：

- 大标题
- 留白
- 图片与文字交错
- 时间切片
- 章节感
- 引文
- 小型标签
- 城市印章

避免：

- 大量阴影
- 过多彩色按钮
- 强烈渐变
- 过度圆角
- 复杂仪表盘
- 评分、排行榜和热度数字

# 5. 图像风格

图片应强调：

- 城市历史
- 建筑
- 街道
- 书店
- 音乐空间
- 老照片
- 地图
- 纸张和印刷质感

建议：

- 降低饱和度
- 使用暖色调
- 轻微颗粒感
- 避免旅游宣传照式高饱和
- 新旧照片可并置

推荐比例：

- Hero：2.2:1 或 16:7
- 主题卡：4:3
- 站点封面：4:3 或 3:2
- 老照片：3:2
- 时间切片：横向图文卡

# 6. 桌面端总体规范

## 6.1 页面宽度

```text
最大内容宽度：1280–1440px
正文阅读宽度：680–780px
侧栏宽度：260–340px
地图栏宽度：320–420px
```

不要让正文横跨整个屏幕。

## 6.2 网格与间距

推荐 12 栏网格。

```text
1440px 屏幕左右边距：64–96px
1920px 屏幕左右边距：120–180px
卡片间距：16 / 24 / 32px
区块间距：64 / 80 / 120px
```

## 6.3 圆角与阴影

```text
按钮：6–10px
卡片：8–14px
图片：6–10px
输入框：6–8px
```

```css
box-shadow: 0 8px 24px rgba(46, 39, 29, 0.06);
```

避免明显悬浮感和过度圆角。

# 7. 桌面端首页 Landing Page

## 7.1 页面目标

首页应回答：

> 今天，你想怎样认识一座城市？

而不是直接展示大量热门地点。

## 7.2 页面结构

### 顶部导航

左侧：

- Logo
- Slogan

中部：

- 探索城市
- 我的路线
- 收藏
- 主题
- 关于我们

右侧：

- 搜索
- 通知
- 用户头像

### Hero 区

左侧：

```text
今天，
你想怎样
认识一座城市？
```

副标题：

```text
AI 与你一起，探索城市的故事与连接。
```

右侧使用城市天际线、历史建筑、河岸或街道，并可加入淡印章或地图线稿。

### 三种探索模式

桌面端横向三栏：

#### AI 帮我发现

```text
我只有时间和兴趣，
剩下交给 AI。
```

#### 我已有几个目标

```text
告诉 AI 必去的地方，
帮我补全路线。
```

#### 我已有路线

```text
导入或打开路线，
帮我优化体验。
```

每个入口包含图标、标题、一句话说明和箭头，整卡可点击。

### 精选主题

横向卡片或六栏网格：

- 上海 1935
- 武康路文学漫游
- 南京音乐地图
- 上海书店巡礼
- 广州开埠时代
- 北京胡同记忆

每张卡片包含图片、城市、主题标题和主题标签。

### 编辑推荐或引文

```text
城市是一本打开的书，
从不同的页面开始，
会读到不同的故事。
```

# 8. 桌面端 AI 规划工作台

## 8.1 页面目标

该页面不是纯聊天框，而是：

> AI 对话 + 可视化路线约束编辑器。

## 8.2 推荐布局

```text
顶部导航
────────────────────────────
左侧 60%：AI 规划对话
右侧 40%：路线摘要与约束
────────────────────────────
底部或主区域：开始生成
```

## 8.3 左侧对话区

问题按步骤展开：

1. 去哪座城市？
2. 有必须去的地方吗？
3. 更偏向哪些主题？
4. 步行能接受多久？
5. 想要怎样的节奏？
6. 是否需要安排餐厅？

回答尽可能结构化：

- POI 标签
- 多选按钮
- 距离选项
- 节奏选项
- 日期和时间控件

避免连续的大段聊天气泡。

## 8.4 右侧路线摘要

可做成纸张或档案卡片：

```text
路线摘要

城市：南京
模式：完善
时间：一天（约 5 小时）
步行距离：5–10 km

必去地点：
• 先锋书店
• 总统府

兴趣偏好：
文学、建筑、书店
```

特点：

- 可编辑
- 实时更新
- 固定在侧栏
- 可以折叠

## 8.5 生成状态

主按钮：

```text
开始生成路线
```

等待文案：

```text
正在翻阅这座城市……
正在寻找与你兴趣有关的故事……
正在整理一条适合今天的路线……
```

# 9. 桌面端 City Reader 路线阅读页

## 9.1 页面定位

它不是传统路线详情页，而是：

> 城市阅读器。

## 9.2 顶部信息区

包含：

- 路线标题
- 城市与主题
- 总距离
- 总时长
- 站点数
- 分享
- 收藏
- 更多
- 城市印章

示例：

```text
书页与旧城之间
南京 · 文学漫游

6.8 km · 预计 5 小时 · 8 个站点
```

## 9.3 阅读 / 导航切换

```text
阅读模式 | 导航模式
```

阅读模式：

- 地图窄栏
- 内容宽栏
- 主题内容展开

导航模式：

- 地图宽栏
- 下一站信息
- 内容收起

## 9.4 地图区域

桌面端：

- 左侧或右侧固定窄栏
- 可折叠
- 可拖动调整宽度
- 低饱和米色底
- 深绿路线
- 圆形站点编号
- 不显示过多地图默认图标

## 9.5 路线时间轴

每站包含：

- 编号
- 时间
- 地点名
- 必去 / 推荐 / 可选
- 简介
- 图片
- 主题标签
- 下一段交通
- 展开阅读
- 替换 / 删除 / 收藏

## 9.6 时间切片卡片

```text
时间切片：1935

这里曾经是什么
后来发生了什么
今天还能看到什么
```

视觉：

- 浅沙色背景
- 老照片
- 档案棕标题
- 小型印章
- 细边框
- 可展开

## 9.7 推荐阅读 / 聆听

### 推荐阅读

```text
《南京城记》
叶兆言
```

### 推荐聆听

```text
肖邦《夜曲》
建议步行途中播放
```

不建议默认自动播放音乐。

## 9.8 沿途可选点

```text
沿途可选点

某独立书店
增加步行 4 分钟

某历史建筑
增加步行 7 分钟
```

操作：

- 加入路线
- 设为备用
- 忽略
- 查看推荐理由

# 13. 手机端 City Reader

## 13.1 默认阅读模式

顶部：

- 路线标题
- 距离与时间
- 主题标签

正文：

- 当前站点
- 时间切片
- 推荐阅读
- 下一站

地图：

- 默认折叠
- 点击后全屏展开

## 13.2 导航模式

显示：

- 当前所在位置
- 下一站
- 距离
- 预计步行时间
- 地图
- 打开高德导航

底部固定操作：

- 已到达
- 跳过
- 查看故事
- 临时调整

## 13.3 站点卡片

每站建议只展示：

- 时间
- 名称
- 80–120 字简介
- 一张图片
- 一个主要主题
- 展开阅读

避免一次展示过多历史内容。

# 14. 响应式断点

```css
mobile: 0–767px
tablet: 768–1023px
desktop: 1024–1439px
wide: 1440px+
```

Desktop → Tablet：

- 三栏变两栏
- 地图缩窄
- 精选主题卡数量减少
- 规划页右侧摘要变窄

Tablet → Mobile：

- 双栏改单栏
- 地图折叠
- 摘要进入抽屉
- 时间轴图片改为卡片顶部
- 顶部导航改为菜单

# 16. 动效原则

动效应克制、安静。

推荐：

- 淡入
- 纸张滑出
- 地图缓慢展开
- 时间轴节点依次出现
- 印章轻微盖印效果
- 卡片展开时高度过渡

时长：

```text
150–300ms
```

避免：

- 弹跳
- 夸张缩放
- 快速旋转
- 大面积视差

---

# 17. 无障碍与易用性

必须确保：

- 正文对比度足够
- 正文字号不低于 16px
- 交互区域不小于 44px
- 标签不能只靠颜色区分
- 地图有文字替代
- 图片有 alt
- 支持键盘操作
- 支持减少动效
- 深色模式后续独立设计

---

# 18. 夜间模式预留

夜间模式暂不作为首版重点，但设计 Token 应预留。

建议方向：

- 深炭黑背景
- 暖米白文字
- 暗橄榄绿强调色
- 低亮度地图
- 图片适当压暗

不要简单做颜色反相。

---

# 19. 禁止事项

不要：

- 把三个桌面页面拼成一个横向展示板
- 把手机界面直接放大当网页
- 使用大量蓝色链接和按钮
- 采用旅游 App 常见的高饱和渐变
- 用“评分、热榜、爆款”作为核心视觉
- 让地图永久占满大部分屏幕
- 把 AI 规划页做成纯聊天机器人
- 一次展示太多长文本
- 使用过重做旧纹理
- 用过多印章和装饰抢夺内容

# 21. 最终设计判断标准

看到页面时，用户应该感到：

- 这是一款有文化气质的产品
- 我是在阅读城市，而不是刷攻略
- AI 在协助我，而不是替我做所有决定
- 地图清晰但不过度抢眼
- 内容值得停下来慢慢读
- 电脑端适合规划，手机端适合边走边看

最核心的判断标准是：

> 它是否像一本真正可以行走的城市杂志？

而不是：

> 它是否像一个功能齐全的旅游 App？
