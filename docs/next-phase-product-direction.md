# 城市伴游优先方向

最后更新：2026-07-23

## 产品收拢

Cultural Citywalk 的下一阶段不再以“尽可能多的规划功能”为中心，而是优先做好一段可在现场独立完成的城市文化伴游：路线有明确主题，讲解有资料来源，用户知道下一步该做什么。

首轮只聚焦南京，先准备三条人工审核的编辑稿：近代南京、民国建筑与城市生活、文学与书店。它们当前在探索页标记为“编辑稿待审核”，不能被视为已发布的正式路线；站点顺序、开放信息和深读文案必须经过资料复核后才可发布。

## 已实现的基础

- 首页和探索页聚焦三条南京主题编辑稿；主导航收拢为探索路线、规划路线、我的路线。
- 规划页默认只生成 5–6 个可调整项；餐厅最多保留 3 个候选，并在缺少高德评分时明确说明。
- 途中模式在移动端改为单列阅读，路线、深读、任务、照片和下一步行动可离线读取；已生成深读会写入本地缓存。
- 站点深读使用百度 AI 搜索三组检索，再由 DeepSeek 依据资料生成；页面会显示检索次数、返回量和采用量。
- 现场问答要求登录，沿用同一份站点资料和来源展示；问答记录只存在当前设备。

## 深读质量规则

- 提示词版本为 `route-collaboration-v0.6`，角色为城市文化导游，而非虚构亲历的朋友。
- 资料充分时为 800–1200 字：摘要加四段导览，分别覆盖历史背景、现场空间、人物或人文故事、与今天城市的联系。
- 只有官方、学术或权威媒体资料可以支持事实；文化旅游文章只可支持明确标注的“相传”或“一种流传说法”。
- 检索资料不足时生成较短的资料有限版，不以编造内容补足长度。
- 每站固定两项可选现场任务，任务不计分，也不以“闯关”命名。

## 人工审核清单

每条正式路线发布前，人工确认：

1. 4–6 个体验站点均为清楚、可抵达的文化站点；
2. 路线能用一句话解释主题主线；
3. 每个核心站点都有可访问的资料来源和审核日期；
4. 深读中的年代、人物、引语、传说和动态信息均符合来源等级；
5. 高德步行、开放时间、预约和休息点经过出发日复核。

## 小规模真实使用

邀请 5–10 位用户完成真实行走。通过条件：多数用户无需额外说明即可走完路线、愿意再次使用、能说出路线主题；出现严重事实、开放信息或安全问题时停止扩展并先修复。

暂不进入社交社区、路线交易、全城市自动覆盖、音频导览和第三方餐厅点评聚合。
